# ゲーム別得点記録機能

## プロダクト概要

- **課題**: 現在の試合分析フォームでは、試合全体の技術別得点を一括入力する形式になっており、ゲームごとの得点推移を記録・把握できない。卓球の試合は複数ゲーム（通常5ゲームマッチ）で構成されるため、ゲーム単位の分析が重要である。
- **ターゲットユーザー**: T.T.LOG を利用して試合分析を行う卓球プレーヤー・コーチ
- **コア価値提案**: ゲームごとに得点を記録し、スコアボードにリアルタイムで反映することで、試合の流れを正確に把握できる試合記録体験を提供する。

## 機能一覧（Features）

### Sprint 1: ゲーム終了・次ゲーム遷移の基本フロー

- F1: ゲーム終了ボタンの表示 -- 「試合を分析する」ボタンの上に「N ゲーム目終了」ボタンを表示する。初期状態は「1ゲーム目終了」。最大5ゲーム目終了まで表示される（match_format に応じて上限が変わる）。
- F2: ゲーム終了時のデータ保存 -- ゲーム終了ボタンを押すと、現在のゲームの技術別得点データ（Score）を Game に紐づけて保存する。Game レコードに game_number、player_score（選手の合計得点）、opponent_score（対戦相手の合計得点）を記録する。
- F3: 次ゲーム入力フォームの表示 -- データ保存後、同じ試合分析フォームを再表示する。日付・大会名・選手名・対戦相手名・メモは前回入力値を保持する。技術別得点の各フィールド（score, lost_score）は 0 にリセットする。
- F4: スコアボードへのゲーム結果反映 -- 次ゲームのフォーム表示時、スコアボードの「ゲーム数」の下にある「ー」プレースホルダーに、保存済みゲームの得点（例: 「11-8」）を表示する。ゲーム勝数（playerWins / opponentWins）を更新する（そのゲームで得点が多かった選手に +1）。
- F5: ゲーム番号の進行 -- ゲーム終了ボタンのラベルが「1ゲーム目終了」→「2ゲーム目終了」→ ... と自動的にインクリメントされる。ボタンのラベルは現在入力中のゲーム番号を反映する。
- F6: 得点数のリセット -- 次ゲーム表示時、スコアボードの「得点数」表示（playerPoints / opponentPoints）を 0 にリセットする。新しいゲームの入力を開始するクリーンな状態にする。
- F7: 最終ゲーム後の試合分析送信 -- 最大ゲーム数（5ゲーム目）の終了ボタンを押した後、または任意のゲーム終了後に「試合を分析する」ボタンを押すと、全ゲームの累計データで試合を保存し、AI 分析を開始する。途中のゲームで「試合を分析する」を押した場合は、現在入力中のゲームも含めて保存する。

### Sprint 2: ゲーム別分析の強化

- F8: ゲーム別得点推移の表示 -- 試合詳細（show）画面で、ゲームごとの得点推移を一覧表示する。各ゲームの技術別得点・失点をテーブルまたはカードで表示する。
- F9: ゲーム別スコアの詳細確認 -- 試合一覧や詳細画面で、各ゲームのスコア（例: 11-8, 9-11, 11-6）を確認できる。
- F10: AI分析へのゲーム別データ提供 -- ChatgptService に渡すデータにゲーム別の得点情報を含め、ゲームごとの傾向分析（例:「3ゲーム目以降フォアドライブの失点が増加」）を可能にする。

### Sprint 3: 入力体験の向上

- F11: ゲーム入力の取り消し（Undo） -- 直前に保存したゲームデータを取り消して、再入力できる機能。誤って「ゲーム終了」を押した場合のリカバリー手段。
- F12: ゲーム途中保存（下書き） -- ゲーム入力途中でブラウザを閉じても、入力中のデータが失われないよう一時保存する機能。
- F13: 試合形式の選択 -- 3ゲームマッチ / 5ゲームマッチ / 7ゲームマッチなど、試合形式をユーザーが選択できる。選択に応じてゲーム終了ボタンの上限とスコアボードの表示枠数が変動する。

## ユーザーストーリー（主要機能のみ）

- **US1**: 卓球プレーヤーとして、1ゲーム目の得点記録が終わったら「1ゲーム目終了」ボタンを押してデータを保存し、すぐに2ゲーム目の入力を開始したい。なぜなら、試合中または試合直後にテンポよくゲームごとの記録を取りたいからだ。

- **US2**: 卓球プレーヤーとして、次のゲーム入力画面に進んだとき、日付・大会名・選手名などは再入力せず、得点フィールドだけがリセットされた状態で表示されてほしい。なぜなら、同じ試合の情報を何度も入力するのは手間だからだ。

- **US3**: 卓球プレーヤーとして、スコアボードに過去のゲームスコア（例: 11-8）とゲーム勝数が表示されてほしい。なぜなら、現在の試合状況を把握しながら次のゲームの記録を取りたいからだ。

- **US4**: 卓球プレーヤーとして、全ゲームの記録が終わったら「試合を分析する」を押して AI 分析を受けたい。なぜなら、ゲームごとの傾向も含めた総合的なアドバイスがほしいからだ。

## スコープ外（Out of Scope）

- リアルタイムスコア入力（1点ずつ加算するライブスコアリング UI） -- 今回は技術別の得点集計フォームをゲーム単位で区切る機能に限定する
- 対戦相手とのオンラインスコア共有
- 過去の試合データのゲーム別分割（既存データのマイグレーション）
- ゲーム中のタイムアウト記録
- 得点のリアルタイムグラフ描画（Sprint 2 のテーブル表示に限定）

## 成功指標

- **定量**: ゲーム終了ボタンを使って2ゲーム以上記録した試合の割合が、機能リリース後1ヶ月で新規試合の50%以上
- **定量**: 試合入力の完了率（フォーム表示 → 分析完了）が従来比で低下しないこと（ゲーム分割による離脱を防ぐ）
- **定性**: ユーザーが「ゲームごとの傾向がわかるようになった」と感じること
- **定性**: ゲーム間の遷移がスムーズで、入力の手間が増えたと感じないこと

## 提案する優先順位

### 1位: #94 ChatGPT APIをgpt-5miniにバージョンアップ
**理由:**
- **#103（worker非同期停止）の前提条件**になっている。gpt-5miniで応答速度が30秒以内になるか確認が必要
- APIバージョンアップにより、コスト削減・レスポンス改善が期待できる
- コアの価値（AI分析）に直結する変更
- 変更範囲が比較的限定的（`ChatgptService`周辺）

### 2位: #103 workerでの非同期を停止
**理由:**
- #94の完了後、応答速度を計測して判断できる
- **Herokuのworker dynoコスト削減**に直結（月額の固定費削減）
- 同期処理に切り替えればSidekiq/Redisの依存も減り、インフラがシンプルになる

### 3位: #108 開発環境をDockerに戻す
**理由:**
- 開発効率の土台。早めに安定させておくことで、以降のタスクがスムーズに進む
- 他の機能開発に影響しないインフラ系タスク

### 4位: #78 不要なファイル・コード削除
**理由:**
- コードの見通しが良くなり、以降の機能追加がやりやすくなる
- #95（サーブレシーブ統合）が完了済みなので、関連する不要コードも洗い出しやすい

### 5位: #100 分析詳細ページに得点数と失点数の本数も表示
**理由:**
- ユーザー体験の改善（既存機能の強化）
- 比較的小規模な変更で完了できる
- 既存のデータを活用するだけなのでDB変更なし

### 6位: #101 分析の際にゲーム数と得点数がわかる得点板を設置

- #100と合わせて進めると効率的

### 7位: New UI/UX強化（全体デザインのモダン化）

- 既存ページの改善（#100, #101）が完了した状態を土台にデザインを整えられる

## 依存関係の整理

```
#94 (API バージョンアップ)
  └→ #103 (worker停止判断)

#108 (Docker環境) → 以降の全開発タスクの土台

#78 (コード整理) → #97, #77 の着手前に完了推奨

#100, #101 → UI/UX強化(New) → #97 → 得点推移表(New) → #77
  └ 既存改善完了 → デザイン基盤確立 → enum確定 → ラリーテーブル設計 → ラリーデータ活用分析
```

## まとめ

大きな方針としては:
1. **まずインフラ・コスト最適化**（#94 → #103）で運用コストを下げる
2. **開発環境の安定化**（#108）で開発効率を確保
3. **コード整理**（#78）で見通しを良くする
4. **既存機能の改善**（#100, #101）でユーザー体験を向上
5. **UI/UX全体のモダン化**（UI/UX強化）でデザイン基盤を確立し、以降の新機能を美しく作る
6. **enum確定**（#97）でデータモデルを安定させる
7. **新データモデル・UI**（得点推移表）でラリーデータを蓄積できる基盤を作る
8. **大規模分析強化**（#77）はラリーデータを活用して最後に実装
